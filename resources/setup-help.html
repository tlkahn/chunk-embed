BGE-M3 Embedding Model

The BGE-M3 model (~2 GB) is required for generating embeddings.

  • Automatic: The model is downloaded automatically the first time you run an embed operation. No action needed.
  • Manual: Clone or copy the model into the local_bge_m3/ directory at the project root: 
    git clone https://huggingface.co/BAAI/bge-m3 local_bge_m3

Editor ($VISUAL / $EDITOR)

An editor is needed to open source files from search results at the correct line number.

Set one of these environment variables in your shell profile (~/.zshrc, ~/.bashrc, ~/.config/fish/config.fish, etc.):

# Pick one:
export EDITOR=code      # VS Code
export EDITOR=nvim      # Neovim
export EDITOR=vim       # Vim
export EDITOR=emacs     # Emacs
export EDITOR=subl      # Sublime Text

For GUI editors that detach from the terminal, set $VISUAL instead (it takes priority over $EDITOR):

export VISUAL="code --wait"

After editing, reload your shell or restart the app.

PostgreSQL + pgvector

A PostgreSQL database with the pgvector extension is required for storing and searching embeddings.

1. Install PostgreSQL

# macOS (Homebrew)
brew install postgresql@17
brew services start postgresql@17

# Ubuntu / Debian
sudo apt install postgresql postgresql-contrib

2. Install pgvector

# macOS (Homebrew)
brew install pgvector

# From source (any platform)
git clone https://github.com/pgvector/pgvector.git
cd pgvector
make && sudo make install

3. Create the database and enable the extension

createdb chunk_embed
psql chunk_embed -c "CREATE EXTENSION IF NOT EXISTS vector;"

4. Verify

Use the Test Connection button on the Setup tab to confirm everything works. The default connection URL is:

postgresql://localhost/chunk_embed

Set the DATABASE_URL environment variable to use a different database.
